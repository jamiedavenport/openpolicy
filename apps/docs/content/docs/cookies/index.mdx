`@openpolicy/cookie-banner` is a [shadcn registry](https://ui.shadcn.com/docs/registry) item that gives you a fully styled, consent-aware cookie banner. Two packages are involved:

- **`@openpolicy/cookie-banner`** â€” the shadcn component that provides the `CookieBanner` and `CookiePreferences` UI
- **`@openpolicy/react`** â€” manages consent state via the `<OpenPolicy>` provider and `useCookies()` hook

## Install

Add the shadcn component and its runtime dependencies:

```sh
bunx shadcn@latest add @openpolicy/cookie-banner
bun add @openpolicy/react @openpolicy/sdk
```

The shadcn command copies `CookieBanner` and `CookiePreferences` into your project's component directory. `@openpolicy/react` provides the provider and consent state that the components read from.

## Configure cookies

Add a `cookie` section to your `openpolicy.ts` config. The categories you define here are surfaced in the preferences panel:

```ts
// openpolicy.ts
import { defineConfig } from "@openpolicy/sdk";

export default defineConfig({
  company: {
    name: "Acme Inc.",
    legalName: "Acme Corporation",
    address: "123 Main St, Springfield, USA",
    contact: "privacy@acme.com",
  },
  cookie: {
    effectiveDate: "2026-01-01",
    cookies: {
      essential: true,
      analytics: true,
      functional: false,
      marketing: false,
    },
    consentMechanism: {
      hasBanner: true,
      hasPreferencePanel: true,
      canWithdraw: true,
    },
    jurisdictions: ["us", "eu"],
  },
});
```

See the [openpolicy.ts reference â†’](/docs/reference/openpolicy-config#cookie) for all available `cookie` fields.

## Add the provider

Wrap your root layout with `<OpenPolicy config={...}>`. The provider reads your cookie config, stores consent decisions in `localStorage`, and makes them available to all child components via React context:

```tsx
// app/layout.tsx (Next.js) or main.tsx (Vite)
import { OpenPolicy } from "@openpolicy/react";
import openpolicy from "@/openpolicy";

export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        <OpenPolicy config={openpolicy}>
          {children}
        </OpenPolicy>
      </body>
    </html>
  );
}
```

## Add the components

Import `CookieBanner` and `CookiePreferences` from wherever shadcn copied them (typically `@/components/ui/cookie-banner`), then place them inside the provider:

```tsx
import { OpenPolicy } from "@openpolicy/react";
import { CookieBanner, CookiePreferences } from "@/components/ui/cookie-banner";
import openpolicy from "@/openpolicy";

export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        <OpenPolicy config={openpolicy}>
          {children}
          <CookieBanner />
          <CookiePreferences />
        </OpenPolicy>
      </body>
    </html>
  );
}
```

**`<CookieBanner />`** â€” the initial consent prompt shown to new visitors. Offers "Accept all", "Accept necessary", and a link to open preferences. Disappears once the user makes a choice.

**`<CookiePreferences />`** â€” a modal panel listing each category with a toggle. Opens when the user clicks "Preferences" in the banner or is triggered programmatically via `useCookies().open("preferences")`. Lets users update their consent at any time.

### Props

Both components read all state from the nearest `<OpenPolicy>` provider. Neither requires any props.

| Component | Props |
|---|---|
| `<CookieBanner />` | None required |
| `<CookiePreferences />` | None required |
